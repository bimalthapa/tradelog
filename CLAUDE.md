# TradeLog

Read `TECHNICAL.md` and `TASKS.md` before doing anything else, including asking clarifying questions.

After reading, state:
- The last completed task and next task up from TASKS.md
- Any open questions from TECHNICAL.md §10 relevant to the next task

## Required skills — invoke BEFORE touching code

For every implementation task, invoke these skills in order:
1. `superpowers:brainstorming` — before adding any new feature or functionality
2. `superpowers:writing-plans` — when the task has multiple steps (most tasks do)
3. `superpowers:test-driven-development` — before writing any production code

For bugs: use `superpowers:systematic-debugging` instead of guessing.

Skipping any of these is a process violation. No exceptions.
